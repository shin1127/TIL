##  n % 3 == 0 と等価な処理

各桁の数の総和が3の倍数であればよい
<--> 100k + 10 n + 1 m.. における、 k + n + mの解が3の倍数であればよい
与えられた数が3桁なら、ぜいぜい取りうる範囲は1から27だが（100から999）

9999999999のように、桁の数が多いととんでもないことになるため、
再帰処理を用いて各桁の総和を減らしていく

最終的に1桁まで縮めることで、その数が3, 6, 9のいずれかかそうでないかという判定方法で等価な処理を行うことができるはずだ。
